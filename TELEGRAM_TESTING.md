## Проверка Telegram Web App и бота

Ниже — минимальные шаги, чтобы подключить бота, Web App и хостинг на Vercel и проверить текущий интерфейс магазина.

### 1. Создать бота и токен

1. Откройте Telegram и найдите бота `@BotFather`.
2. Отправьте команду `/newbot` и следуйте инструкциям.
3. В конце BotFather вернёт строку вида:  
   `1234567890:AA...` — это **Bot Token**.
4. Сохраните токен и впишите его в файл `.env` проекта:

   ```env
   NUXT_BOT_TOKEN=ВАШ_ТОКЕН_ОТ_BOTFATHER
   NUXT_MANAGER_CHAT_ID=ID_МЕНЕДЖЕРА
   ```

5. `NUXT_MANAGER_CHAT_ID` — это числовой Telegram ID менеджера (можно взять у себя, если пока тестируете сами). Удобнее всего получить ID через бота `@userinfobot` или подобного — он вернёт ваш numeric ID.

### 2. Подготовить Web App URL

Сейчас фронтенд Nuxt разворачивается как обычный веб‑сайт. Его URL нам понадобится в двух местах:

- в BotFather — как **web app** для кнопки;
- в настройке вебхука Telegram — как `https://<HOST>/api/webhook`.

Если вы ещё не деплоили на Vercel, сначала можно проверять локально (см. ниже шаг про ngrok), но конечная схема для продакшена выглядит так:

- публичный URL: `https://ВАШ-ПРОЕКТ.vercel.app`
- Web App URL: тот же URL;
- Webhook URL: `https://ВАШ-ПРОЕКТ.vercel.app/api/webhook`.

### 3. Зарегистрировать Web App кнопку в BotFather

1. Откройте чат с `@BotFather`.
2. Выберите своего бота: `/mybots` → нужный бот → **Bot Settings**.
3. Перейдите в раздел **Menu Button** → **Web App** (в разных версиях интерфейса это может называться `Edit Menu Button`).
4. Введите:
   - **Название**: например, `Открыть магазин`;
   - **Web App URL**: публичный адрес вашего Nuxt приложения (локально через ngrok или версельный URL).
5. Сохраните изменения.

После этого при нажатии на кнопку в чате с ботом будет открываться ваш Nuxt Web App (витрина).

### 4. Настроить вебхук Telegram (после деплоя)

> Этот шаг будет иметь смысл, когда реализованы роуты `server/api/webhook.post.ts` и `server/api/order.post.ts` по ТЗ.

1. Определите итоговый публичный URL приложения, допустим:

   ```text
   https://ВАШ-ПРОЕКТ.vercel.app
   ```

2. Сформируйте URL для выставления вебхука:

   ```text
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/setWebhook?url=https://ВАШ-ПРОЕКТ.vercel.app/api/webhook
   ```

3. Откройте этот URL в браузере или запросите через curl/Postman. В ответ Telegram должен вернуть JSON с `"ok": true`.

4. После этого:
   - команда `/start` в чате с ботом будет обрабатываться вашим `webhook.post.ts`;
   - нажатия менеджера на inline‑кнопки статусов заказов тоже будут приходить в этот эндпоинт.

### 5. Проверка работы фронтенда через Telegram

После того как:

- в `.env` указаны корректные `NUXT_BOT_TOKEN` и `NUXT_MANAGER_CHAT_ID`;
- Nuxt‑приложение задеплоено и открывается по публичному URL;
- в BotFather настроена Web App кнопка;

можно пройти такой сценарий:

1. Откройте чат с вашим ботом в Telegram.
2. Нажмите кнопку **«Открыть магазин»** (Web App).
3. Проверьте:
   - витрина с корейским меню;
   - добавление/удаление позиций, счётчики на карточках и в корзине;
   - открытие корзины в виде модального **bottom‑sheet**;
   - кнопку **«Оформить заказ»** (пока без реального отправления на `/api/order`).
4. Обновите страницу внутри Web App (или закройте и заново откройте): состояние корзины должно **восстанавливаться** из `localStorage`.

## Деплой на Vercel

Ниже — базовые шаги, чтобы задеплоить проект на Vercel и использовать его как хостинг для Telegram Web App и вебхука.

### 1. Подготовка репозитория

1. Инициализируйте git в корне проекта (если ещё не сделали):

   ```bash
   git init
   git add .
   git commit -m "Initial Nuxt Telegram shop"
   ```

2. Создайте репозиторий на GitHub / GitLab / Bitbucket и привяжите его:

   ```bash
   git remote add origin <SSH_ИЛИ_HTTPS_URL>
   git push -u origin main
   ```

### 2. Создать проект на Vercel

1. Зайдите на `https://vercel.com` и войдите (GitHub / GitLab / email).
2. Нажмите **New Project** → выберите репозиторий с этим Nuxt‑проектом.
3. Vercel сам определит фреймворк **Nuxt 3**. Обычно ничего менять не нужно:
   - **Framework Preset**: Nuxt.js;
   - **Build Command**: `npm run build` (по умолчанию);
   - **Output Directory**: `.output/public` или автоматически (Nuxt 3 + Nitro).

### 3. Настроить переменные окружения на Vercel

В настройках проекта на Vercel:

1. Откройте **Settings → Environment Variables**.
2. Добавьте:

   - `NUXT_BOT_TOKEN` — токен бота от BotFather;
   - `NUXT_MANAGER_CHAT_ID` — Telegram ID менеджера.

3. Сохраните и **Redeploy** проект, чтобы переменные попали в рантайм.

### 4. Подтянуть продакшн‑URL в Telegram

1. После успешного деплоя Vercel даст URL вида:

   ```text
   https://ваш-проект.vercel.app
   ```

2. Этот URL:
   - укажите в BotFather как Web App URL;
   - используйте в вебхуке как `https://ваш-проект.vercel.app/api/webhook`.

### 5. (Опционально) Локальное тестирование вебхуков через ngrok

Пока приложение не на Vercel, можно отлаживать вебхуки локально:

1. Запустите Nuxt:

   ```bash
   npm run dev
   ```

2. Поднимите ngrok (или аналог) на порт 3000:

   ```bash
   ngrok http 3000
   ```

3. Возьмите выданный публичный URL вида `https://xxx.ngrok.io` и выставьте вебхук:

   ```text
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/setWebhook?url=https://xxx.ngrok.io/api/webhook
   ```

4. Теперь все апдейты Telegram будут прилетать в ваш локальный `server/api/webhook.post.ts`.

- подключить вебхук Telegram к вашему backend‑роуту `/api/webhook` и проверить весь сценарий заказа, когда бэкенд будет реализован.
