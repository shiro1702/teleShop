# Рекомендации: скорость витрины города на Vercel (SSR, редиректы, API)

Документ зафиксирован для последующей реализации. Контекст: главная платформы редиректит на `/{defaultCitySlug}` (например `/ulan-ude`), страница списка ресторанов сейчас подгружает данные клиентским `fetch` после гидрации.

## Текущее состояние (ориентир)

- Редирект `/` → `/{defaultCitySlug}`: `middleware/redirect-city.global.ts` (`navigateTo`).
- `pages/[city_slug]/index.vue`: `watch(..., { immediate: true })` вызывает `loadCityAndShops()` — параллельно **`/api/cities?slug=…`** и **`/api/shops?city_slug=…`** через `fetch` в браузере.
- При SSR в разметке остаётся состояние загрузки («Загрузка ресторанов…»), пока клиент не выполнит запросы.

## Задачи по приоритету

### 1. Перенести загрузку на сервер (SSR)

- Заменить клиентский `fetch` на **`useAsyncData` / `useFetch`** (или общий серверный хелпер), чтобы в первом HTML-ответе для `/ulan-ude` уже были **название города и список ресторанов**.
- Переносить **оба** источника данных (`cities` + `shops`), иначе выигрыш неполный.

### 2. Убрать дублирующие запросы к БД

- `/api/shops` уже резолвит город по `city_slug` (таблица `cities`), `/api/cities` — отдельный запрос по тому же slug.
- Варианты: один эндпоинт «city + shops», либо общая server-утилита, которую вызывают SSR и при необходимости API.

### 3. Редирект `/` на edge (Vercel)

- Рассмотреть **`vercel.json` redirects** или **`routeRules` в Nuxt** с `redirect`, чтобы ответ был **307/308 с CDN** без лишней работы приложения для корня домена.

### 4. Кэширование публичных ответов API (по политике свежести)

- Короткий `Cache-Control` / SWR для списков города/ресторанов, если допустима задержка обновления (например 30–120 с). Учесть требования к актуальности витрины.

### 5. Прочие улучшения

- **Prefetch** переходов на карточки ресторанов (`NuxtLink`, где уместно).
- **Изображения**: явные размеры, lazy, при необходимости `@nuxt/image`.
- **Тяжёлый UI** только в `ClientOnly` / lazy-компонентах (карта уже через lazy — сохранять подход).
- **Скрипты в `nuxt.config`** (Telegram / MAX web app): подключать точечно на нужных страницах, если не нужны на витрине города — меньше парсинга JS на первом экране.
