# План работ по фестивалю: От Демо до Продакшена

Для успешной продажи идеи организаторам и последующего запуска мы разбиваем работу на два этапа. Это позволит не тратить ресурсы на сложный бекенд, пока нет 100% подтверждения, и сфокусироваться на "вау-эффекте" для первой презентации.

---

## Этап 1: Демонстрационная версия (Продажа идеи организаторам)
**Цель:** Показать визуальную концепцию, вау-эффекты и простоту использования для гостя и повара.

**Что делаем:**
1. **Структура данных "Фестиваль":** 
   - Завести базовую сущность "Фестиваль" в БД (можно реализовать как отдельный `city_id` = "Фестиваль" или флаг `is_festival` на уровне группы ресторанов). 
   - Настроить изолированную выдачу корнеров (гость видит только участников фестиваля).
2. **Верстка клиентского приложения (Моки и Заглушки):**
   - **Сториз и виджеты:** Создать блок с расписанием и "Пульсом фестиваля" ("Съедено 1500 бургеров", "Сэкономлено 400 часов"). Данные можно захардкодить или обновлять простейшим ручным конфигом.
   - **Витрина корнеров:** Сверстать фестивальный дизайн меню.
   - **Вкладка с Ачивками:** Отрисовать UI (иконки, прогресс-бары "Гастро-турист", "Флэш", "Легенда Фестиваля"). Пока без реального трекинга в бекенде.
3. **Верстка B2B-экранов:**
   - **Тапалка для кухни:** Экран с моковыми заказами и большой кнопкой "Готово".
   - **Экран "Лидерборд" (для ТВ/сцены):** Заглушка с топом ресторанов ("Хит фестиваля", "Народная любовь").
4. **Легенда для презентации:**
   - Показываем организаторам путь гостя: Скан QR ➔ Видит сториз/афишу ➔ Выбирает еду ➔ Получает "ачивку" ➔ Забирает еду без очереди.
   - Делаем акцент на сбор базы данных (post-event пуши) и геймификацию (см. `festival-gamification-and-retention.md`).

---

## Этап 2: Production (Подготовка к реальному фестивалю)
**Цель:** Оживить функционал, обеспечить отказоустойчивость и реальный процессинг заказов.

**Что делаем:**
1. **Реальный флоу заказа:**
   - Интеграция реальных меню участников фестиваля.
   - QR-роутинг: сканирование QR-кода должно корректно открывать миниапп/веб с привязкой к фестивальной зоне.
   - Оплата (СБП/эквайринг), реальное пробитие чека и прозрачные расчеты с точками.
2. **Инфраструктура B2B (Оживление):**
   - Рабочая "тапалка" для поваров с реальными сокетами/поллингом новых заказов.
   - Рабочая кнопка "Пауза приема заказов" (меняет статус корнера на `busy` и скрывает его с витрины гостей).
   - Дашборд организатора: реальный подсчет GMV, `orders_total`, `avg_prep_time` и пиковых нагрузок.
3. **Реализация базовой геймификации:**
   - Подключение счетчиков для 2-3 ключевых ачивок (например, "Гастро-турист" по количеству разных корнеров, "Флэш" по времени забора заказа).
   - Формирование лидерборда "Хит фестиваля" и "Народная любовь" на основе реальных данных из БД.
4. **Подготовка базы к Post-Event:**
   - Сбор сегментов пользователей (кто, где ел, что заказывал) для автоматизации push-уведомлений (Этапы 1-3 из стратегии удержания).
   - Логика генерации уникальных промокодов, привязанных к профилю конкретного пользователя.

## Этап 3: Post-Festival (Переход в городскую среду)
**Цель:** Конвертировать фестивальный трафик в постоянных пользователей агрегатора и дать ресторанам B2B-ценность.

**Что делаем:**
1. **Трансформация клиентского интерфейса:**
   - Снятие "фестивального режима": переключение главной страницы агрегатора обратно на общую городскую ленту (переход от изолированной зоны к `/:city_slug`).
   - Конвертация фестивальных ачивок в постоянные статусы профиля (например, добавление "Золотой рамки" или бейджа "Мэр фуд-корта" в профиле клиента).
2. **Запуск кампаний удержания (Push-маркетинг):**
   - Отправка "Итогов фестиваля" (Wrapped-формат для соцсетей) в понедельник.
   - Запуск таргетированных пушей с промокодами от заведений, где пользователь совершал заказы (Среда-Четверг).
   - Обучающая рассылка о том, как использовать приложение в городе (Пятница).
3. **Развертывание B2B-кабинетов (Дашборды):**
   - Открытие расширенной аналитики для ресторанов (тепловые карты заказов, скорость отдачи, retention).
   - Открытие расширенного дашборда для организаторов фестиваля (сводный GMV, пиковые нагрузки, база для следующего года).
4. **Умная лента ресторанов:**
   - Внедрение алгоритмов сортировки на главной странице города на основе фестивальных данных ("Быстрые рестораны" на основе "Турбо-отдачи", "Топ-рейтинг" на основе "Народной любви").
