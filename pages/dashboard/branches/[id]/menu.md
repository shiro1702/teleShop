# /dashboard/branches/:id/menu

- Настройка меню конкретного филиала: **локальные цены**, **скрытие позиций**, **стоп-лист по товарам**, *(план)* **стоп по ингредиентам**.

**Параметры позиции** (размер / объём / количество с **своей ценой** на вариант) задаются в **глобальном** меню на карточке товара; на уровне филиала по умолчанию действуют те же варианты и цены, пока не введены отдельные филиальные оверрайды (см. `pages/dashboard/menu.md`). Модификаторы остаются надстройкой к выбранному варианту.

## Стоп и видимость по товару (текущая реализация в дашборде)

В UI (`menu.vue`) для каждой активной позиции глобального меню доступны:

| Поле | Назначение (продуктовая семантика) |
|------|-------------------------------------|
| **Своя цена** | `price_override` в `restaurant_product_overrides`; если пусто — используется базовая цена бренда. |
| **В стоп-листе** | `is_in_stop_list`: позиция **не заказывается** на этом филиале. Поведение на витрине — см. раздел **«UX витрины: стоп и скрытие»** ниже. |
| **Скрыть** | `is_hidden`: позиция **не показывается** в каталоге для этого филиала (как будто её нет в меню точки). Отличается от стопа: стоп = видно, но нельзя заказать; скрыть = не видно вовсе. |

## UX витрины: стоп и скрытие

### Скрытые позиции (`is_hidden`)

- В **каталоге и поиске** не показывать карточку (как будто позиции нет в меню этой точки).
- Если товар уже **в корзине** до скрытия — при оформлении сработает серверная проверка (см. `pages/[city_slug]/[tenant_slug]/cart.md`).

### Стоп-лист (`is_in_stop_list`) — рекомендуемый режим по умолчанию

Цель: гость **понимает**, что блюдо в меню есть, но сейчас недоступно, и не думает, что приложение сломалось.

1. **Карточка в сетке каталога остаётся на месте**, не пропадает из категории.
2. **Визуальный “серый” стоп (приглушение):** сниженная непрозрачность карточки (ориентир **~0.55–0.7**) и/или лёгкий grayscale, без обязательного полного Ч/Б — сохранять читаемость бренда.
3. **Бейдж** на карточке: короткий текст **«Временно нет»** или **«Нет в наличии»** (единый словарь на витрине).
4. **Первичная кнопка:** не «В корзину», а неактивная **«Недоступно»** / disabled с понятной подписью; контраст disabled — доступный (a11y).
5. **Модалка товара:** по тапу открывается, вверху или у CTA — то же состояние стопа; состав и параметры можно показывать для прозрачности, добавление в корзину заблокировано.

**Категории:** не прятать целиком категории только потому, что все позиции в стопе (если категория не пустая по смыслу меню); иначе навигация «прыгает». Допустимо показывать заголовок категории с приглушенными карточками.

### Опция в настройках бренда (post-MVP)

- **«Позиции в стопе не показывать в каталоге»** — визуально ближе к `is_hidden`, но управляется флагом стопа на филиале. Минус: гость не видит, что блюдо существует. Включать только осознанно.

### Стоп по ингредиенту

- То же отображение, что и стоп по позиции: карточка приглушена, бейдж «Временно нет» (при желании уточнение в модалке не обязательно в MVP).
- То же отображение, что и стоп по позиции: карточка в “сером” приглушении, бейдж «Временно нет» (при желании уточнение в модалке не обязательно в MVP).

**Что ещё не доведено до конца в продукте**

- Публичный API витрины (`/api/products` и выбор ресторана) должен **подмешивать** `restaurant_id` и применять `is_hidden` / `is_in_stop_list` / `price_override` так же, как дашборд. Пока это зафиксировано как обязательный шаг MVP в роадмапе `pages/dashboard/menu.md`.
- В дашборде нет (до конца не продуман UX): **поиска**, **массового выделения**, **истории**, **таймера стопа**, а также сценариев массового управления стопом по товарам (`is_in_stop_list`) и скрытием (`is_hidden`) с предпросмотром “сколько позиций затронуто”.

## Стоп по ингредиентам *(план)*

Когда внедрён справочник ингредиентов и матрица `product_ingredients`, на этой же странице филиала (или на подстранице) добавляется блок **«Стоп по ингредиентам»**: переключатель по каждому релевантному ингредиенту. Включение стопа помечает **все товары**, в составе которых есть этот ингредиент, как недоступные к заказу на данном филиале (детали и модель БД — `pages/dashboard/menu/ingredients.md`).

**Совместное действие с стопом по товару:** рекомендуемое правило для MVP — позиция недоступна, если **стоп по позиции ИЛИ стоп по любому ингредиенту из её состава**. Тогда кухне достаточно выключить один ингредиент (например закончилась моцарелла), не кликая десятки пицц.

## Недочеты / риски

- Не описаны конфликты между брендовым меню и филиальными override (кроме явных полей в `restaurant_product_overrides`).
- Нет контроля массовых изменений цен и отката правок.
- Нет описанного процесса публикации изменений (черновик/публикация) для глобального меню.
- Витрина должна совпадать по смыслу с дашбордом для `is_hidden` / `is_in_stop_list` / цены.
