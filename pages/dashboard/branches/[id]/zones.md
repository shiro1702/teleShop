# /dashboard/branches/:id/zones

- Управление зонами доставки филиала (полигоны GeoJSON, тарифы, приоритет).

## Реализовано

- CRUD зон в дашборде: список, создание, редактирование, удаление (`/api/dashboard/branches/:id/zones`, см. `pages/dashboard/branches/[id]/zones.vue`).
- Поля зоны: название, `polygon_geojson` (Feature или Polygon), стоимость доставки, порог бесплатной доставки, минимальная сумма заказа, **priority** (при одинаковой цене доставки — больше важнее), флаг активности.
- **Редактор зоны на карте** (`components/dashboard/DeliveryZoneMapEditor.vue`): Leaflet + тайлы OpenStreetMap (`https://{s}.tile.openstreetmap.org/...`), панель рисования и редактирования полигона (Leaflet.draw). Атрибуция «© OpenStreetMap» на карте. Полигон на карте синхронизируется с полем GeoJSON; при несовместимом с MVP GeoJSON (MultiPolygon, дыры, невалидный JSON) показывается тост, текст вручную не перезаписывается картой — правка в блоке «Расширенное: GeoJSON вручную».
- **Ограничение MVP (UI карты):** один внешний контур `Polygon` (первое кольцо), без дыр и без MultiPolygon в интерактивном редакторе; сложные кейсы — только через JSON.
- Правила на витрине (см. также `POST /api/delivery-resolve`, `server/utils/resolveDeliveryForPoint.ts`):
  - по адресу с координатами подбираются все зоны всех филиалов бренда, в которые попадает точка;
  - сортировка кандидатов: **меньшая стоимость доставки**, затем **больший priority**, затем стабильный tie-break;
  - выбирается первый **открытый** филиал в этом порядке; если самый выгодный филиал закрыт — берётся следующий открытый (клиенту показывается ненавязчивое сообщение);
  - если подходящих открытых филиалов нет или адрес вне зон — оформление доставки блокируется с понятной причиной.
- Финальный расчёт зоны и `restaurant_id` при наличии `address.lat` / `address.lon` в `POST /api/order` выполняется на сервере (клиентские поля не подменяют гео-логику).
