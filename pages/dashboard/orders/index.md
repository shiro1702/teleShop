# /dashboard/orders

- Список заказов для операционной работы менеджеров и владельца.

## Что уже можно делать

- Смотреть список заказов.
- Применять фильтры: период, бренд, город, филиал, статус.
- Выполнять действия: открыть заказ, сменить статус, экспорт.

## Канбан в рамках /dashboard/orders
Канбан в этом разделе предназначен для **операционной работы менеджеров** (диспетчеризация по статусам заказов).

На странице поддерживаются режимы отображения (переключение по табам):
- `Таблица` (текущий MVP-режим)
- `Канбан по статусам` (колонки соответствуют `order.status`)

Учёт разных режимов работы и филиалов:
- канбан должен строиться с учетом текущего `fulfillment_type` (например `delivery`, `pickup`, `dine-in`, `qr-menu`, `showcase-order`)
- канбан должен быть отфильтрован по выбранному `филиалу` (то есть `shop_id`), чтобы не смешивать очереди разных точек

## Недочеты / риски

- Не зафиксированы пустые состояния (нет заказов, нет доступа, ошибка загрузки).
- Не описана приоритизация статусов и SLA-индикаторы для "проблемных" заказов.
- Нет явного описания пагинации, сортировки и массовых операций.
- Не описана стратегия ограничений WIP, лимитов по колонкам и журнал переносов (для drag-and-drop).
