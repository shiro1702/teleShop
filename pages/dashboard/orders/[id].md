# /dashboard/orders/:id

- Подробная карточка конкретного заказа.

## Что уже можно делать

- Смотреть полные данные заказа.
- Менять статус заказа.
- Оставлять комментарии менеджера и смотреть историю событий.
- Проверять лог доставки и контактные данные клиента.

## Недочеты / риски

- Не описана стратегия блокировок при одновременном редактировании.
- Нет явной валидации переходов между статусами.
- Не выделены критичные действия (отмена, возврат) с подтверждением.

### Канбан-совместимость
Если в `/dashboard/orders` включается канбан по статусам (колонки по `order.status`), карточка заказа должна:
- корректно отражать текущий статус
- отображать историю переносов/изменений статуса (включая источник: вручную или через канбан)
- валидировать переходы статусов по заранее заданной матрице (`allowedStatusTransitions`)
