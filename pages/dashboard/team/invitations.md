# /dashboard/team/invitations

- Управление приглашениями в организацию.
- Основной сценарий: Owner приглашает менеджера по email/ссылке.

## Что должно быть тут

- Форма "Пригласить менеджера":
  - email,
  - роль (на MVP обычно только `Manager`),
  - набор базовых permissions.
- Список инвайтов:
  - `pending` / `accepted` / `expired` / `revoked`,
  - дата создания и срок действия,
  - кто отправил.
- Действия:
  - повторно отправить,
  - отозвать инвайт,
  - скопировать ссылку.

## Логика авторизации и принятия инвайта

- Приглашение формирует токен и ссылку вида `/invite/:token`.
- На `/invite/:token`:
  - проверка валидности токена;
  - если пользователь не авторизован -> редирект на `/register` или `/login` с сохранением token;
  - после авторизации -> привязка пользователя к организации с ролью из invite.

## Ограничения (MVP)

- Только `Owner` может создавать/отзывать приглашения.
- Один активный pending-инвайт на email в рамках одной организации.
- Просроченные токены не принимаются.

## Что уже можно делать

- Отправлять приглашения менеджерам по email/ссылке.
- Отслеживать состояния инвайтов: `pending`, `accepted`, `expired`, `revoked`.
- Повторно отправлять, отзывать и копировать invite-ссылку.

## Недочеты / риски

- Не описаны лимиты частоты повторной отправки.
- Нет явной защиты от brute-force подбора invite-токена.
- Не описан UX для массового контроля и фильтрации инвайтов.
