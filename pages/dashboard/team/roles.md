# /dashboard/team/roles

- Экран настройки уровней доступа в организации.
- Цель: зафиксировать кто и что может делать в дашборде.

## Что должно быть тут

- Таблица ролей:
  - `Owner` — полный доступ;
  - `Manager` — ограниченный доступ по permissions.
- Матрица permissions по разделам:
  - заказы (просмотр/смена статуса),
  - филиалы (просмотр/создание/редактирование),
  - меню (просмотр/редактирование),
  - настройки организации (просмотр/редактирование webhook и токенов),
  - команда (приглашать/удалять менеджеров).
- Переключатели прав для роли `Manager`.
- История изменений прав (кто изменил, когда).

## Базовые правила (MVP)

- Только `Owner` может:
  - назначать/снимать `Owner`,
  - выдавать критичные permissions,
  - удалять менеджеров из организации.
- `Manager` по умолчанию:
  - видит заказы и филиалы,
  - не может управлять составом команды и критичными настройками.

## Техническая логика (ожидаемая)

- Источник прав: `profiles.role` + `permissions` (jsonb) или отдельная membership-таблица.
- Проверки на сервере обязательны (нельзя полагаться только на UI).
- UI скрывает/блокирует действия без разрешения, но итоговая проверка всегда в API.
