# /dashboard

- Главная точка входа во внутренний кабинет партнера (Owner/Manager).
- На этой странице не редактируются сущности напрямую; она нужна для ориентации и быстрых переходов.

## Что должно быть на странице

- Краткий блок "Быстрые действия" с кнопками:
  - `/dashboard/orders` — перейти к заказам;
  - `/dashboard/branches` — открыть список филиалов;
  - `/dashboard/settings/organization` — открыть настройки организации.
- Виджет "Подключенные филиалы" вынесен на отдельную страницу `/dashboard/branches`.
- На перспективу (после MVP): компактные KPI-карточки "заказы сегодня", "выручка", "проблемные статусы".
- После стабилизации MVP: добавить графики на главной дашборда:
  - выручка и количество заказов по дням/неделям;
  - распределение заказов по статусам;
  - топ филиалов по выручке/количеству заказов за период.

## Навигация отсюда в другие разделы

- На странице `/dashboard` оставить только короткие CTA:
  - Заказы -> `/dashboard/orders`;
  - Филиалы -> `/dashboard/branches`;
  - Настройки -> `/dashboard/settings/organization`.
- В `layouts/dashboard.vue` хранить "глобальную" навигацию для всех внутренних страниц:
  - Заказы -> `/dashboard/orders`;
  - Филиалы -> `/dashboard/branches`;
  - Новый филиал -> `/dashboard/branches/new`;
  - Настройки -> `/dashboard/settings/organization`.
- Итого: кнопки на странице = контекстные быстрые действия; верхнее меню = постоянная навигация.

## Логика доступа

- Страница доступна только авторизованному пользователю.
- Если пользователь не авторизован -> редирект на `/login`.
- Если нет контекста организации/филиала -> редирект на `/onboarding`.
