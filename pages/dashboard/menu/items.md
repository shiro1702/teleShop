# /dashboard/menu/items

- Раздел управления позициями меню.

## Что должно быть на странице

- Список товаров с фильтрами:
  - категория;
  - поиск по названию.
- Карточка товара:
  - название;
  - описание;
  - базовая цена;
  - изображение;
  - принадлежность к категории;
  - блок состава (ингредиенты);
  - модификаторы/опции.
- Блок модификаторов в карточке товара:
  - выбор групп модификаторов;
  - настройка обязательности;
  - порядок отображения;
  - override доплаты для конкретного товара (если нужно).
- Действия:
  - создать товар;
  - редактировать;
  - удалить/архивировать.

## User flow (MVP)

1. Пользователь открывает `/dashboard/menu/items`.
2. Нажимает "Создать товар".
3. Заполняет обязательные поля (название, категория, цена).
4. Сохраняет товар.
5. Публикует изменения меню.

## User flow модификаторов (товар)

1. Менеджер открывает товар.
2. Подключает группу `Размер` и добавляет опции `20см/30см/40см`.
3. Подключает `Соус` как `single-choice`.
4. Добавляет boolean-опции: `Добавить сыр`, `Убрать лук`.
5. Сохраняет товар и публикует меню.
6. На витрине клиент видит эти модификаторы в модалке товара.

## User flow ингредиентов (товар)
1. Менеджер открывает товар.
2. В блоке **«Состав (ингредиенты)»** выбирает ингредиенты из справочника и задаёт порядок.
3. Для каждой строки состава при необходимости включает `may_contain` (и `quantity_note`, если используется).
4. Сохраняет товар и публикует меню.
5. На витрине в модалке товара отображается состав ингредиентов.

## Что должен увидеть клиент

- Карточка товара:
  - индикатор "есть опции".
- Модалка:
  - обязательный выбор размера;
  - выбор одного соуса;
  - доп. переключатели (сыр/лук).
- Модалка (дополнительно):
  - список ингредиентов в порядке `sort_order`;
  - строка “Может содержать …” по флагу `may_contain`.
- Корзина:
  - выбранные модификаторы под названием товара;
  - итоговая цена с учетом доплат.
- Заказ:
  - те же модификаторы в составе каждой позиции.

## Критичные риски (решаем в MVP)

- [ ] Валидация цены, названия и категории на сервере.
- [ ] Исключить публикацию товара без обязательных полей.
- [ ] Зафиксировать поведение при удалении товара, который уже в заказах.
- [ ] Ограничить редактирование цен и удаление товаров по ролям.
- [ ] Не допускать добавление товара в корзину без required-модификаторов.
- [ ] Фиксировать выбранные модификаторы в структуре order-item без потери данных.

## Некритичные риски (чеклист на потом)

- [ ] Массовое изменение цен (bulk update).
- [ ] Импорт/экспорт товаров через CSV.
- [ ] Версионирование карточки товара и откат изменений.
- [ ] Матричные модификаторы (например размер + тесто + бортик) с зависимостями.
