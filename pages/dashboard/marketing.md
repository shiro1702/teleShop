# /dashboard/marketing (Маркетинг, промокоды и бонусы)

Раздел `/dashboard/marketing`: управление промокодами и настройками бонусной программы (без «магазина за баллы» в текущей версии).

## Реализовано (MVP)

### Промокоды
- Создание, редактирование, удаление записей в таблице `shop_promo_codes` (код в БД в верхнем регистре, без пробелов).
- Типы: `percent`, `fixed`, `free_item` (подарок подставляется на сервере в состав заказа с ценой 0; для варианта позиции задаётся UUID опции параметра).
- Ограничения: минимальная сумма заказа (по сумме товаров до скидки), `starts_at` / `ends_at`, `max_uses_total`, `max_uses_per_user` (только для авторизованных клиентов с профилем; без авторизации лимит «на пользователя» недоступен).
- **Одна скидка:** других автоматических скидок в коде пока нет; при появлении «счастливых часов» и т.п. не суммировать с промокодом без отдельного продукта-решения.

### Бонусы (1 бонус = 1 ₽)
- Настройки в `shop_loyalty_settings`: процент начисления от суммы товаров после промо (до списания бонусов), максимальный процент заказа, оплачиваемый бонусами, сгорание (флаги и дни; автоматическое списание неактивности — позже).
- Поля: приветственный бонус, день рождения, бонус за отзыв — хранятся в настройках; **начисление за отзыв** не выполняется до появления модуля отзывов на платформе (см. TODO ниже).
- Начисление бонусов за заказ: после успешной **онлайн-оплаты** (webhook YooKassa), идемпотентно по записи в `loyalty_ledger` с `reason = earn_order`.
- Списание бонусов при оформлении заказа; баланс в `shop_customer_balances`, журнал в `loyalty_ledger`.

### Не реализовано (бэклог)
- **Магазин за баллы** (отдельный каталог товаров за бонусы).
- **Грейды лояльности** (новичок / любитель / босс).
- **Таргетинг промо** только на категорию или товар (фаза 2; в схеме можно заложить `targeting jsonb` позже).
- **Отзывы на платформе:** нет сущности отзывов; после внедрения можно начислять `review_bonus_amount` по правилам ресторана.

API дашборда: `/api/dashboard/marketing/promo-codes`, `/api/dashboard/marketing/loyalty-settings`.

## Целевая эволюция (Rule Engine)
Долгосрочно возможен движок правил с триггерами (сумма корзины, тип доставки, время, история клиента) и наградами — см. архивные идеи в истории документа; MVP ограничен перечисленным выше.
