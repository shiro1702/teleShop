# /dashboard/stories (Управление Сториз и Маркетинговыми Кампаниями)

- Раздел для создания и управления интерактивными "Сториз" (подобно Instagram) для ресторана.
- Позволяет настраивать таргетированные маркетинговые кампании, управлять контентом (слайдами) и анализировать их эффективность.

## 1. Функционал и возможности (Уровень бизнес-логики)

- **Создание кампаний (Кружочков):** Владелец может создать кампанию, задать ей название (текст под кружочком), загрузить превью-картинку и установить таймер жизни (с какого по какое число показывать).
- **Управление слайдами:** Внутри кампании можно добавлять неограниченное количество слайдов (фото или короткие видео). Для каждого слайда задается длительность показа (обычно 5-15 секунд) и порядок.
- **Интерактивные действия (Диплинки):** К каждому слайду можно привязать кнопку с целевым действием:
  - `add_to_cart` — добавить конкретный товар в корзину (с указанием ID товара и количества).
  - `apply_promo` — применить промокод (с указанием самого промокода).
  - `open_category` — перейти в определенную категорию меню.
  - `none` — просто информационный слайд без кнопки.
- **Таргетинг (Персонализация):** Владелец может выбрать, кому показывать сториз:
  - Всем пользователям (включая неавторизованных гостей).
  - По полу (например, только женщинам на 8 Марта).
  - По дню рождения (например, за 7 дней до дня рождения).
  - По истории заказов (например, "Спящие" клиенты, которые не заказывали более 30 дней, или VIP-клиенты с количеством заказов > 10).
- **Выбор места размещения (Placement):**
  - `top_bar` — классическое размещение в виде кружочков над каталогом.
  - `catalog_grid` — интеграция прямо в сетку товаров (в виде вертикального баннера).
- **Перспектива (Агрегатор):** В будущем появится возможность покупать места для своих сториз на главной странице агрегатора (кросс-маркетинг).

## 2. UI/UX Дашборда (Как это выглядит для менеджера)

- **Список кампаний:** Таблица или сетка карточек с текущими, запланированными и прошедшими кампаниями. Статусы: "Активна", "Черновик", "Завершена".
- **Конструктор кампании:**
  - **Шаг 1: Основные настройки.** Название, превью, даты проведения, выбор места размещения (`top_bar` или `catalog_grid`).
  - **Шаг 2: Настройка таргетинга.** Визуальный конструктор правил (выпадающие списки: "Пол", "День рождения", "Давность заказа").
  - **Шаг 3: Редактор слайдов.** Drag-and-drop интерфейс для загрузки медиа, сортировки слайдов. Форма для настройки кнопки (выбор типа действия и привязка товара/промокода через поиск по базе).
- **Аналитика кампании:**
  - Просмотры (по каждому слайду).
  - Клики по кнопкам (CTR).
  - Конверсии (сколько товаров добавлено в корзину из сториз, сколько промокодов применено).

## 3. Архитектура БД (Supabase)

Для реализации функционала требуются следующие таблицы:

### `story_campaigns` (Кампании / Кружочки)
- `id` (uuid, primary key)
- `shop_id` (uuid, foreign key -> tenants)
- `title` (text) — Название под кружочком.
- `preview_url` (text) — Картинка самого кружочка.
- `placement` (enum: `top_bar`, `catalog_grid`) — Где выводить.
- `is_active` (boolean) — Ручной выключатель.
- `valid_from`, `valid_until` (timestamp) — Таймер жизни кампании.
- `targeting` (jsonb) — Правила таргетинга (например: `{"genders": ["female"], "birthday_within_days": 7, "min_orders_count": 0}`).

### `story_slides` (Экраны внутри кружочка)
- `id` (uuid, primary key)
- `campaign_id` (uuid, foreign key -> story_campaigns)
- `order` (integer) — Порядок (1, 2, 3...).
- `media_url` (text) — Видео или фото.
- `duration` (integer) — Время показа в секундах.
- `action_type` (enum: `add_to_cart`, `apply_promo`, `open_category`, `none`) — Что делает кнопка.
- `action_payload` (jsonb) — Данные для действия (например: `{"item_id": "123", "qty": 1}` или `{"code": "MARCH8"}`).

### `story_views` (Аналитика просмотров)
- `id` (uuid, primary key)
- `slide_id` (uuid, foreign key -> story_slides)
- `user_id` (uuid, nullable, foreign key -> auth.users)
- `viewed_at` (timestamp)

## 4. Расширенные интерактивы в Сториз (Планы на развитие)
Помимо базовых кнопок "Добавить в корзину" и "Применить промокод", планируется внедрение продвинутых механик вовлечения:

- **Опросы / Голосования (Polls):** Клиенты могут голосовать (например, "Какую пиццу добавим: 🍕Мясную / 🧀Сырную"). Это дает бизнесу обратную связь, а проголосовавшим можно позже отправить пуш-уведомление со скидкой на выбранный продукт.
- **Геймификация:** Механики вроде "Сотри защитный слой" или "Колесо фортуны" прямо внутри сториз. Клиент тапает по экрану, стирает слой и получает случайный подарок (промокод или товар).
- **Сбор обратной связи (NPS):** Оценка недавнего заказа прямо в сториз (от 1 до 5 звезд).
- **Бронирование столика:** Кнопка, открывающая форму бронирования или перекидывающая в WhatsApp ресторана.
- **Шеринг (Реферальная программа):** Кнопка "Поделиться с другом", генерирующая уникальную ссылку. При заказе друга оба получают бонусы.
