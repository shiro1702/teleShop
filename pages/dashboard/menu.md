# Управление меню

В системе заложена модель **Глобальное меню + Локальные переопределения**.
- **Глобальное меню (`/dashboard/menu`)**: управляется владельцем или глобальным менеджером. Здесь создаются категории, товары, модификаторы, задаются базовые цены и привязки к iiko (`external_id`).
- **Локальное меню филиала (`/dashboard/branches/:id/menu`)**: управляется менеджером конкретной точки. Здесь можно только управлять стоп-листами (временно скрыть товар) и переопределять цены для конкретного филиала.

## Что должно быть на странице `/dashboard/menu`

- Быстрые ссылки на разделы:
  - `/dashboard/menu/categories` — управление категориями (создание, сортировка, привязка iiko_id);
  - `/dashboard/menu/items` — управление позициями (базовая цена, фото, состав, привязка к категории и модификаторам);
  - `/dashboard/menu/modifiers` — управление группами модификаторов (размеры, добавки).
- Общий статус меню:
  - черновик / опубликовано;
  - дата последнего обновления;
  - кто последний редактировал.
- Базовые действия:
  - создать категорию;
  - создать товар;
  - отправить изменения в публикацию.

## User flow (MVP)

1. Пользователь открывает `/dashboard/menu`.
2. Выбирает блок "Категории" или "Позиции меню".
3. Вносит изменения.
4. Проверяет предпросмотр и валидацию.
5. Публикует изменения.
6. Система фиксирует действие в audit log.

## Роутинг для модификаторов

- `GET /dashboard/menu/modifiers`:
  - список групп модификаторов (например "Размер", "Добавки", "Соусы");
  - фильтрация по типу: single/multi/boolean.
- `GET /dashboard/menu/modifiers/:id`:
  - редактирование группы и опций;
  - ограничения выбора (min/max, required).
- Привязки:
  - из `/dashboard/menu/items/:id` (или модалки редактирования товара) — выбор доступных групп;
  - опционально из `/dashboard/menu/categories/:id` — дефолтные модификаторы на категорию.

## Модификаторы: базовая модель

- На уровне товара:
  - размер пиццы (`20см/30см/40см`) как группа `single-choice`, обязательный выбор;
  - добавить сыр (`boolean` или `single option` с доплатой);
  - убрать лук (`boolean` без доплаты);
  - соус (`single-choice`) из набора предложенных.
- На уровне категории:
  - дефолтные модификаторы для всех товаров категории (например соусы для категории "Бургеры");
  - товар может переопределить дефолтные правила.

## UI в dashboard (редактирование меню)

- Страница `Модификаторы`:
  - создание группы модификаторов;
  - выбор типа (`single`, `multi`, `boolean`);
  - опции группы: название, доплата, сортировка, активность;
  - ограничения: `required`, `min_select`, `max_select`.
- Страница `Позиции меню`:
  - блок "Модификаторы товара";
  - выбор существующих групп;
  - локальный override (например другая цена для "30см" у конкретной пиццы).

## UI клиента (витрина)

- В карточке товара:
  - короткие бейджи активных модификаторов ("Размер", "Соус").
- В модалке товара:
  - обязательные модификаторы валидируются до добавления в корзину;
  - single-choice рендерится radio/select, multi-choice — checkbox, boolean — toggle.
- В корзине клиента:
  - каждая позиция показывает выбранные модификаторы;
  - доплаты модификаторов входят в цену позиции;
  - при редактировании позиции сохраняется конфигурация модификаторов.
- В оформленном заказе:
  - в каждой строке заказа фиксируются выбранные модификаторы;
  - менеджер видит их в заказе без потери структуры.

## Supabase (предлагаемая структура)

Для поддержки иерархии, сортировки и интеграции с POS (iiko) потребуется расширить текущую схему:

- `categories` (новая таблица):
  - `id`, `shop_id`, `name`, `sort_order`, `is_active`, `external_id` (для iiko).
- `products` (обновление текущей таблицы):
  - заменить `category text` на `category_id uuid references categories(id)`;
  - добавить `sort_order integer default 0`;
  - добавить `external_id text` (для iiko).
- `modifier_groups`:
  - `id`, `shop_id`, `name`, `scope` (`item|category|both`), `selection_type` (`single|multi|boolean`),
  - `required`, `min_select`, `max_select`, `is_active`, `sort_order`, `external_id`.
- `modifier_options`:
  - `id`, `group_id`, `name`, `price_delta`, `is_default`, `is_active`, `sort_order`, `external_id`.
- `product_modifier_groups`:
  - `product_id`, `group_id`, `is_required_override`, `min_select_override`, `max_select_override`.
- `category_modifier_groups`:
  - `category_id`, `group_id`.
- `restaurant_product_overrides` (управление стоп-листами и ценами на уровне филиала):
  - `shop_id`, `restaurant_id`, `product_id`, `price_override integer`, `is_hidden boolean`, `is_in_stop_list boolean`.
- `order_item_modifiers`:
  - `order_item_id`, `group_id`, `option_id` (nullable для boolean), `label`, `price_delta`.
