# /:city_slug/:tenant_slug/checkout

- Checkout бренда в контексте города.
- Филиалы, зоны и стоимость доставки берутся из связки `brand + city`.
- В checkout не показываем ссылку «Все рестораны», чтобы не отвлекать от оформления.

## Онлайн-оплата (YooKassa / Т-Банк)

- Кнопка "Оплатить онлайн" запускает только серверный сценарий:
  1. фронт отправляет запрос на сервер (`/api/checkout/create`);
  2. сервер резолвит merchant-конфиг `shop` (в будущем с `restaurant` override);
  3. сервер создает платеж у провайдера и возвращает `confirmation_url`;
  4. клиент редиректится на платежную страницу провайдера.

- Возврат на страницу `success`/`pending` — только UX-этап.
- Источник истины по оплате: webhook провайдера на бэкенд платформы.
- После подтвержденного webhook:
  - заказ получает `payment_status=paid`,
  - заказ уходит в операционный поток кухни/уведомлений.

## Негативные сценарии (обязательные)

- Пользователь закрыл вкладку после оплаты -> заказ все равно перейдет в `paid` по webhook.
- Пользователь отменил платеж -> checkout показывает повтор оплаты.
- Провайдер прислал дубль webhook -> обработка идемпотентна, статус не дублируется.

## Валидация заказа

Серверные проверки корзины (стоп-лист, цена, параметры, модификаторы, комбо, каналы) — чеклист в `pages/[city_slug]/[tenant_slug]/cart.md`, раздел **«Проверки при отправке корзины / создании заказа»**.
