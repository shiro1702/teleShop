# /:city_slug

- Техническая проработка (избранное, повтор заказа, чипы «настроение», отзывы): [docs/AGGREGATOR_UX_FEATURES_RU.md](../../docs/AGGREGATOR_UX_FEATURES_RU.md).

- Главная агрегатора в конкретном городе.
- Сценарии: выбор ресторана, поиск еды, поиск ресторана, фильтры, проверка адреса, быстрый повтор заказа.
- Обязательный дисклеймер: доставку выполняет сам ресторан.
- Использовать `city_slug` для фильтрации выдачи.
- Это точка возврата из ресторанных сайтов по ссылке «Все рестораны».

## Цель главной

- За первые 10 секунд помочь голодному пользователю принять решение и перейти к заказу.
- Сократить путь до корзины и оплаты (минимум кликов для повторяемых сценариев).
- Повысить retention и среднюю частоту заказов за счет лояльности и ситуативных триггеров.
- Монетизировать трафик через платные размещения (сториз, баннеры, flash-акции, приоритет в ленте).

## Блоки главной (порядок сверху вниз)

### 1) Шапка: адрес + режим получения

- Показывать текущий адрес доставки.
- Добавить свитчер `Доставка <-> С собой` в шапке.
- При `С собой` приоритетно открывать карту ресторанов с условиями самовывоза.

### 2) Сториз (монетизация)

- Оставить текущий верхний блок сториз как premium-слот.
- В агрегаторе сториз использовать для акций ресторанов по городу.

### 3) Повторить заказ (быстрые действия)

- Показывать только авторизованным пользователям при наличии истории.
- Формат: горизонтальная карусель карточек с фото блюда, названием, рестораном, ценой, CTA `В корзину`.
- Главная задача: добавить типовой заказ в 1 тап без перехода в меню ресторана.

### 4) Лояльность: "До подарка остался 1 шаг"

- Вынести прогресс штампов/лояльности на главную отдельным блоком.
- Формат: прогрессбар по ресторану/кампании + явный оффер (например, "Купите еще 1 кофе и получите бесплатно").
- Блок работает как триггер возврата и переключения выбора в пользу недостающего шага до награды.

### 5) Баннеры / Акции / Счастливые часы

- Свайпаемая карусель промо-блоков.
- Для flash-акций использовать таймер обратного отсчета (`Осталось: HH:MM:SS`) для FOMO-эффекта.
- Использовать для загрузки "мертвых часов" (например, 15:00-17:00) и платного промо размещения.

### 6) Ситуативные подборки (вместо скучных категорий)

- Вместо сухих "Пицца/Суши/Бургеры" показывать сценарные коллекции:
  - `На компанию`
  - `Полезное / ПП`
  - `От похмелья`
  - `До 500 рублей`
- Задача: помогать выбрать по состоянию и бюджету, когда у пользователя нет конкретного запроса.

### 7) Привезут быстрее всего

- Отдельная карусель ресторанов/блюд с акцентом на ETA (например, `У вас через 25 минут`).
- Ранжирование по геопозиции пользователя, SLA кухни и доступности курьеров.
- Этот блок должен быть заметным для сценария "очень голоден, важна скорость".

### 8) Что заказывают рядом (социальное доказательство)

- Формат: ненавязчивый live-feed или список "топ-5 блюд в вашем районе за сегодня".
- Пример событий: "Только что заказали ...".
- Цель: усиливать доверие и подталкивать к популярным позициям.

### 9) Основная лента ресторанов

- Бесконечный скролл как базовый слой каталога.
- Ранжирование: сначала платное продвижение (с четкой маркировкой), затем качество по рейтингу/скорости/релевантности.

## Режим "С собой" и карта

- При выборе `С собой` показывать карту с пинами ресторанов.
- На пине отображать полезный короткий оффер: минимальная цена или скидка за самовывоз.
- Основной сценарий: "по пути домой" + экономия на доставке.

## Умная выдача и ограничения интерфейса

- Не показывать все блоки одновременно в полном объеме: использовать персонализацию и ротацию.
- Блоки `Повторить заказ` и `Лояльность` скрывать, если нет данных пользователя.
- На первом экране держать фокус на 2-3 наиболее конверсионных элементах для конкретного пользователя.
- Обязательно избегать визуальной перегрузки: один главный акцент на экран, остальное — вторично.
